# Q04 — Use the SELinux role

## Question

> Create a playbook called `/home/student/ansible/playbooks/selinux.yml` that:
>
> - Runs on all managed nodes
> - Uses the `rhel-system-roles.selinux` role
> - Configures SELinux policy as `targeted`
> - Sets SELinux state as `enforcing`

---

## Approach

Install `rhel-system-roles` via dnf on the control node. Copy the example playbook from `/usr/share/doc/rhel-system-roles/selinux/` and strip it down. The role uses a `block/rescue` pattern because applying SELinux changes often requires a reboot — the rescue catches the `selinux_reboot_required` variable, reboots the node, waits for it to return, and re-applies the role.

---

## Solution

### Step 1 — Install the RPM on control

```bash
sudo dnf -y install rhel-system-roles
ansible-galaxy list   # confirm rhel-system-roles.selinux appears
```

### Step 2 — Playbook

File: `playbooks/selinux.yml`

```yaml
---
- hosts: all
  become: true
  become_method: sudo
  become_user: root
  vars:
    selinux_policy: targeted
    selinux_state: enforcing
  tasks:
    - name: Execute SELinux role and catch reboot requirement
      block:
        - name: Include selinux role
          ansible.builtin.include_role:
            name: rhel-system-roles.selinux
      rescue:
        - name: Fail if error is not reboot-related
          ansible.builtin.fail:
            msg: "role failed"
          when: not selinux_reboot_required

        - name: Reboot managed host
          ansible.builtin.reboot:

        - name: Wait for managed host to come back
          ansible.builtin.wait_for_connection:
            delay: 10
            timeout: 300

        - name: Re-apply the selinux role after reboot
          ansible.builtin.include_role:
            name: rhel-system-roles.selinux
```

---

## How to run

> **Important:** Use `ansible-playbook`, NOT `ansible-navigator` for this question.
> The `rhel-system-roles` package installs roles as RPM files, not as a collection.

```bash
cd /home/student/ansible
ansible-playbook playbooks/selinux.yml
```

---

## Verification

```bash
# Check SELinux config file on all nodes
ansible all -m shell -a 'grep ^SELINUX= /etc/selinux/config'
# Expected: SELINUX=enforcing

# Check runtime SELinux status
ansible all -m shell -a 'getenforce'
# Expected: Enforcing
```

---

## Common exam traps

- **`ansible-playbook` not `ansible-navigator`** — RPM-installed roles run with `ansible-playbook`; collection-installed roles run with `ansible-navigator`. Using the wrong command causes the role not to be found.
- **Do not rename the role** — it must be `rhel-system-roles.selinux` exactly. Shortening to `selinux` will fail.
- **The `rescue` block must check `selinux_reboot_required`** — without the `when: not selinux_reboot_required` guard, any non-reboot failure silently triggers a reboot instead of failing.
- **`selinux_policy: targeted` not `selinux_policy: default`** — the exam specifies targeted explicitly.
